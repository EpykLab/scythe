# Plan: Implement 9 AI Authoring Improvements

## Scope and Decisions
- Implement all 9 items in phased milestones with backward-compatible CLI behavior.
- `new --from-intent` v1 is deterministic, local, rule-based (no external model dependency).
- `discover routes` v1 is OpenAPI-first with optional safe live probing.
- Keep Typer and argparse behavior in lockstep for every new flag/command.

## Current Baseline (from repo)
- Command wiring is centralized in `scythe/cli/main.py` (Typer + argparse fallback).
- Existing AI-facing structured output: `check --json`; `run` is text-only today.
- Fixture exists as `fixture serve` with hardcoded routes.
- Tests are `unittest`-style in `tests/test_cli.py`; CI uses `python -m unittest discover -s tests -p "test*.py" -v`.

## Architecture Additions

### New helper modules
- `scythe/cli/diagnostics.py`: shared report/detection payload builders and JSON envelope.
- `scythe/cli/errors.py`: central error taxonomy (`code`, `severity`, `category`, `hint`, `docs`).
- `scythe/cli/output.py`: common text/json renderers for all commands.
- `scythe/cli/fixers.py`: safe autofix actions for `check --fix`.
- `scythe/cli/intent.py`: deterministic intent classifier and template mapping.
- `scythe/cli/discover.py`: OpenAPI parsing + optional probe implementation.
- `scythe/cli/doctor.py`: environment and workflow readiness checks.
- `scythe/cli/snippets.py`: snippet lookup/ranking utilities.
- `scythe/fixtures/profiles.py`: fixture profile definitions and loader.

### New data/assets
- `scythe/snippets/pack.json`: known-good tagged snippets.
- `scythe/fixtures/builtin_profiles/*.json`: fixture route/behavior profiles.
- `scythe/evals/scenarios/*.yaml`: prompt scenarios for nightly eval harness.

### Existing files to extend
- `scythe/cli/main.py`: command/flag plumbing and dispatch.
- `tests/test_cli.py`: integration tests for new commands/flags and parity.
- `README.md`, `docs/AI_TEST_AUTHORING.md`, `docs/README.md`: user-facing docs updates.
- `.github/workflows/`: add nightly eval workflow.

## Delivery Plan

## Milestone 1 - Diagnostics and Repair Foundation
1. Implement `scythe run --json` (Item 2)
   - Add structured run report with summary, per-test results, exit status, diagnostics.
   - Preserve current text output when `--json` is absent.
2. Implement `scythe check --fix` (Item 1)
   - Add safe fixers for canonical issues (missing main guard, missing `--url`, missing `-> int` annotation, compatible versions normalization).
   - Re-run validation after fixes and include `applied_fixes` in JSON output.
3. Expand error taxonomy and hints (Item 9)
   - Replace ad hoc command errors with centralized codes/hints.
   - Use shared codes in `check`, `run`, and fixer failures.

Acceptance criteria
- `run --json` emits machine-readable envelope and deterministic keys.
- `check --fix` can convert representative failing files to passing in one run.
- All command failures include stable taxonomy codes.

Effort: M

## Milestone 2 - Local Authoring Ergonomics
4. Add fixture profiles to `scythe fixture serve` (Item 5)
   - `--profile`, `--list-profiles`, optional `--profile-file`.
   - Keep existing behavior as default profile for backward compatibility.
5. Add snippet pack and lookup command (Item 6)
   - `scythe snippet lookup <query> [--json] [--show <id>]`.
   - Rank by id/tag/token overlap and return deterministic ordering.
6. Add `scythe doctor ai` (Item 8)
   - Validate local setup, project state, command availability, fixture/snippet readiness.
   - Output text by default and JSON with `--json`.

Acceptance criteria
- Fixture routes are profile-driven and test-covered.
- Snippet lookup returns stable results for repeated queries.
- `doctor ai --json` emits actionable remediation hints.

Effort: M

## Milestone 3 - Smart Generation and Route Discovery
7. Add `scythe new --from-intent` (Item 3)
   - Rule-based classifier maps intent text to template kind + starter actions.
   - Add explainability fields (`matched_rule`, `score`) in JSON or console output.
8. Add `scythe discover routes` (Item 4)
   - `discover routes --openapi <file|url> [--probe-base-url ...] [--json]`.
   - Parse spec first; optional probe annotates reachability/status.
   - Probe defaults to safe methods (`GET/HEAD`) and strict timeouts.

Acceptance criteria
- `new --from-intent` is deterministic and overrideable via explicit `--kind`.
- `discover routes --json` returns normalized method/path inventory plus probe annotations.

Effort: L

## Milestone 4 - Continuous AI Quality Measurement
9. Add AI eval harness and nightly tracking (Item 7)
   - Harness executes scenario prompts against generation flow, then runs `check` + `run`.
   - Persist JSON artifact with pass/fail, diagnostics, timing, and trend fields.
   - Add nightly GitHub Actions workflow with artifact upload and regression thresholds.

Acceptance criteria
- Nightly run produces versioned artifact and pass-rate trend.
- Regressions are surfaced by stable error codes and scenario IDs.

Effort: L

## Command Contracts to Add

### Shared JSON envelope
```json
{
  "schema_version": "1.0",
  "command": "run|check|discover|doctor|snippet",
  "timestamp": "ISO-8601",
  "ok": true,
  "data": {},
  "diagnostics": []
}
```

### `run --json`
- `data.summary`: passed/failed/skipped/duration_ms.
- `data.results[]`: test id, status, duration, optional error code/message.

### `check --json --fix`
- Existing diagnostics + `data.applied_fixes[]` (fix id, files, count).

### `discover routes --json`
- Route entries: method, path, operationId/tags, confidence source, optional probe result.

### `doctor ai --json`
- Check entries: id, status, observed/required values, error code, hint.

## Compatibility and Migration Rules
- Additive flags only; existing defaults remain unchanged.
- Keep Typer and argparse parity by routing both into shared internal command functions.
- No destructive live probing defaults; probing is opt-in.
- `check --fix` only applies safe deterministic changes in v1.

## Risks and Mitigations
- Risk: command surface bloat in one file.
  - Mitigation: move logic to helper modules, keep `main.py` as thin adapter.
- Risk: output schema churn breaks AI consumers.
  - Mitigation: versioned envelope + contract tests + additive field policy.
- Risk: `check --fix` alters intent.
  - Mitigation: safe-only fixers and explicit `applied_fixes` reporting.
- Risk: route probing side effects.
  - Mitigation: OpenAPI-first, safe method allowlist, opt-in probing.
