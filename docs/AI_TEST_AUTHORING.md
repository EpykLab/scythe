# AI Test Authoring Contract

This document defines the canonical script contract for Scythe tests generated by AI tools.

## Canonical Contract

A valid Scythe test script should include:

- `COMPATIBLE_VERSIONS` as a list of strings near the top of the file.
- `check_url_available(url)` to preflight the target URL.
- `check_version_in_response_header(args)` when `--gate-versions` is used.
- `scythe_test_definition(args) -> int` that returns an integer exit code.
- `main()` that parses args, validates URL/version gates, and exits with the return code.
- `if __name__ == "__main__": main()` guard.

## AI-Friendly Workflow

Use this loop when creating or updating tests:

1. `scythe init`
2. `scythe new <name> --kind <template-kind>`
   - Optional: `scythe new <name> --from-intent --intent "..." --json`
3. Edit the generated script with project-specific routes/assertions.
4. `scythe check <name> --json`
5. Fix diagnostics and re-run `scythe check` until `"ok": true`.
6. Optionally auto-apply safe fixes with `scythe check <name> --json --fix`.
7. Optionally enforce warning-free scripts with `scythe check <name> --json --strict`.
8. `scythe run --json <name> -- --url <target>`

## Template Kinds

`scythe new` supports the following built-in archetypes:

- `api-journey` (default): Unauthenticated route policy checks using `Journey` and `ApiRequestAction`.
- `api-auth-journey`: Authenticated API checks using `CookieJWTAuth` and CSRF handling.
- `ttp-api`: API-mode TTP example using `LoginBruteforceTTP`.
- `playwright-run`: Run existing pytest-playwright test files with result assertions.
- `playwright-wrap`: Use Playwright's sync API directly in a scythe test.
- `sb-route-matrix`: Stellarbridge-style route policy matrix (public/authz/private).
- `sb-mfa-gate`: Stellarbridge-style MFA non-compliance enforcement check.
- `sb-org-rbac`: Stellarbridge-style organization RBAC check.

These templates follow the same structure and differ only by execution primitives.

## Validator Output

`scythe check --json` emits machine-readable diagnostics:

```json
{
  "path": "/abs/path/to/test.py",
  "ok": false,
  "errors": [
    {
      "code": "missing_test_definition",
      "message": "Missing `scythe_test_definition(args)` function.",
      "hint": "Define this function and return an integer exit code."
    }
  ],
  "warnings": []
}
```

`errors` must be empty for a passing check. `warnings` are advisory.

Use `--strict` to fail when warnings are present.

Use `--fix` to apply safe deterministic repairs before revalidation.

## Run Output

`scythe run --json` emits a versioned envelope with:

- `data.summary` (`passed`, `failed`, `skipped`)
- `data.results[]` status entries
- `diagnostics[]` with error code and hint when execution fails

## Local Fixture Target

For deterministic local iteration:

```bash
scythe fixture serve --host 127.0.0.1 --port 8787
```

This fixture returns:

- Stable `X-SCYTHE-TARGET-VERSION` header.
- CSRF cookie (`__Host-csrf_`) on responses.
- Login endpoint (`/api/v1/auth/login-handler`) with cookie JWT behavior.
- Representative public/private/RBAC/MFA routes used by built-in templates.

Profile options:

- `--list-profiles` to inspect built-in profiles
- `--profile minimal` to select built-in behavior sets
- `--profile-file /path/to/profile.json` for custom route models

## Playwright Integration

Scythe supports two Playwright primitives (requires `pip install 'scythe-ttp[playwright]'` + `playwright install`):

- **`Run(test_file)`**: Execute a pytest-playwright test file as a subprocess and assert on results.
  ```python
  from scythe.playwright import Run
  Run("tests/test_login.py").expect(passed=True)
  ```

- **`Wrap()`**: Use Playwright's sync Page API directly with managed lifecycle.
  ```python
  from scythe.playwright import Wrap
  with Wrap(headless=True) as pw:
      pw.page.goto("https://target.com/login")
      pw.page.fill("#username", "admin")
      pw.page.click("button[type=submit]")
      pw.expect_url_contains("/dashboard")
  ```

- Both are available as Journey Actions: `PlaywrightRunAction` and `PlaywrightWrapAction`.
- Use `playwright-run` or `playwright-wrap` template kinds when generating Playwright-based tests.

## Authoring Notes

- Prefer API mode (`JourneyExecutor(..., mode="API")` or API-mode TTPs) unless browser behavior is required.
- For browser tests, consider Playwright (`scythe.playwright`) as an alternative to Selenium UI mode.
- Keep route expectations explicit (`expected_status`, `expected_result`) per action.
- Use `flush=True` only for requests whose response/session side-effects must be discarded.
- Keep `COMPATIBLE_VERSIONS` up to date and sync it with `scythe db sync-compat <name>`.
